EXPLANATIONS

Test results are based on the code checked out from Git at 7pm EDT each night. Results are evaluated the next morning at 8:45am EDT. The raw test results are saved into a test result archive that contains the results per year, month and day.

Interpretation of Test Results

Notation used in the tables:
  • Red text shows what failed in a test.
  • Green text indicates that a test passed.
  • CAPITALIZED text shows results that have changed.
  • Hover over the failed tests to see the date when it passed last. ?/?/? is shown if the test has never passed.
  • Click on the failed tests, the results or the log file links under the machine names for more information.
The test results are given a simple numeric value for quantitative evaluation:
  • 1.0 - passed
  • 0.5 - result
  • 0.1 - run or rundir
  • 0.0 - compile
The score shown below each day is the success rate of all the tests that were performed that day on all the machines. If the score is red, then either too few tests were done or the success ratio was too low (below 95 per cent). If the score is green, the master Git branch is merged into the stable Git branch for all repositories that have a stable branch. The scores are also saved into the all_scores.txt files in the test result archive for each day.
